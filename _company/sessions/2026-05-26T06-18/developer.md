### 🛠️ 웹사이트 리뉴얼 전 필수 기술 진단 질문지 (Technical Scope Audit)

이 질문들은 크게 **① 아키텍처 범위**, **② 핵심 기능(백엔드)**, **③ 데이터 및 콘텐츠 흐름** 세 가지 축으로 나뉩니다.

#### 🎯 Part 1: 아키텍처 & 범주 정의 (The Big Picture)
*(전체 프로젝트의 근간이 되는 질문입니다.)*

1.  **사이트 종류와 목적:** 이 웹사이트는 어떤 역할을 할 예정입니까?
    *   A. **정적 정보 제공형(Static/Brochure):** 회사 소개, 포트폴리오 등 '정보를 보여주는' 역할만 합니다. (가장 간단)
    *   B. **콘텐츠 관리 시스템(CMS) 기반:** 블로그, 케이스 스터디 등 지속적으로 새로운 글이나 콘텐츠가 올라와야 합니까? (워드프레스 같은 구조 필요)
    *   C. **상호작용/트랜잭션 발생형(Dynamic/Transactional):** 로그인, 결제, 회원가입, 데이터 제출 등 사용자가 *어떤 행동을 하고 시스템이 그 결과를 저장해야 하는 경우*. (**→ 백엔드 전면 개편 필수**)
2.  **예산 및 시간 제약:** 프로젝트에 할당된 개발 예산과 예상 완료 시점(마일스톤)이 있습니까? (이는 기술 스택 선택의 폭을 좁히는 가장 중요한 변수입니다.)

#### ⚙️ Part 2: 핵심 기능 구현 진단 (Backend Dependencies)
*(사용자가 버튼을 눌렀을 때, 서버와 데이터베이스가 작동해야 하는 영역들입니다. 여기에 '예'가 하나라도 있다면 백엔드 개편이 필요합니다.)*

| 기능 | 질문 내용 | 필수 기술 요소 (확인 사항) |
| :--- | :--- | :--- |
| **1. 문의/견적 요청** | 사용자가 이름, 연락처, 원하는 서비스를 입력하고 *데이터를 저장하여* 관리자에게 알림을 보내야 합니까? | 📝 **폼 데이터 수집 API:** (Backend) DB Write / 알림 기능 (Email Service) |
| **2. 결제 모듈 연동** | 서비스 가입비, 컨설팅 비용 등 유료 결제가 발생하여 *결제 기록이 저장*되어야 합니까? | 💳 **PG사(Payment Gateway) API:** (Backend) 트랜잭션 처리 / 상태 관리 로직 |
| **3. 사용자 인증/회원가입** | 고객이 개인 계정을 만들고, 로그인하거나 마이페이지에서 자신의 데이터를 조회해야 합니까? | 🔑 **인증 시스템(Auth):** (Backend) 회원 정보 DB 저장 및 검증 로직 |
| **4. 자동화 워크플로우 실행** | 웹사이트에서 받은 데이터(예: 이미지 파일 여러 개)를 서버로 보내서 AI가 실제로 처리하고, 그 결과를 보여줘야 합니까? | ⚙️ **백그라운드 작업 큐:** (Backend) API Gateway & Worker Queue (API 호출 후 장시간 로직 처리 필요) |

#### 💾 Part 3: 데이터 및 콘텐츠 흐름 진단 (Data Source)
*(사이트가 가져와야 할 정보의 출처를 파악합니다.)*

1.  **포트폴리오 데이터 소스:** 현재 포트폴리오에 올라온 프로젝트 사례는 **① 수동으로 업로드된 파일**입니까, 아니면 **② 특정 내부 시스템(CRM/노션 등)에서 데이터를 연동** 받아와야 합니까? (데이터의 출처가 명확해야 합니다.)
2.  **이미지 및 미디어:** 웹사이트에 사용될 이미지는 외부 AI 툴이나 스톡 이미지 서비스에서 생성되거나 구매하는 방식이 기본입니까? 아니면, **저희 회사 내부 사진작가 팀이 직접 촬영/제공**할 예정입니까? (운영 주체 파악)
